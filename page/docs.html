Документація Yamato

Повна інструкція з торгової платформи — від реєстрації до запуску автоматичних торгових стратегій.

Швидкий старт Торгові боти Безпека FAQ

Yamato об'єднує в собі професійні інструменти для аналізу ринку, автоматизовану торгівлю через ботів, управління портфелем та систему безпеки на рівні фінансових додатків. Платформа розроблена як для початківців, що роблять перші кроки у криптотрейдингу, так і для досвідчених трейдерів, яким потрібен надійний та зручний інструмент.

Основні можливості платформи

Графіки реального часу
Інтерактивний TradingView графік з десятками індикаторів та інструментами технічного аналізу. Дані надходять безпосередньо з Binance без затримок.
Торгові боти
Автоматизована торгівля 24/7 через Binance API. Створюйте ботів з різними стратегіями, тестуйте їх у Paper-режимі перед запуском на реальні кошти.
Управління портфелем
Повний огляд активів з візуалізацією розподілу, відстеження PnL по кожній позиції та історія всіх транзакцій.
Безпека
Двофакторна автентифікація (TOTP), Passkeys (WebAuthn), шифрування API ключів та захищені сесії з cookie httpOnly.
Демо-режим
Віртуальний баланс для безпечного навчання. Всі функції платформи доступні без ризику втрати реальних коштів.
Сповіщення
Миттєві сповіщення в додатку та через Telegram: спрацювання ботів, виконання ордерів, зміни на ринку.

Для кого Yamato?

  • Початківці — демо-рахунок з віртуальними коштами дозволяє навчитися торгувати без фінансового ризику. Інтуїтивний інтерфейс та ця документація допоможуть швидко освоїтись.
  • Активні трейдери — професійний TradingView графік, швидке виконання ордерів та детальна аналітика портфелю для щоденної торгівлі.
  • Алготрейдери — автоматичні торгові боти з підключенням до Binance API, Paper-торгівля для тестування стратегій та детальна статистика по кожному боту.
  • Інвестори — зручне відстеження портфелю, розподіл активів, історія транзакцій та моніторинг ринкових цін у реальному часі.

Як користуватись цією документацією

Зліва розташований зміст з усіма розділами. Натисніть на будь-який пункт для швидкого переходу. Розділи організовані від базових (реєстрація, вхід) до просунутих (боти, ордери, безпека). Якщо ви новачок — читайте по порядку. Якщо шукаєте конкретну інформацію — використовуйте зміст для навігації.

Початок роботи

Цей розділ описує перші кроки на платформі: від створення акаунту до першого входу в дашборд.

Реєстрація

Для створення акаунту перейдіть на сторінку реєстрації:

1
Відкрийте сторінку /register. Ви побачите форму реєстрації з полями для заповнення.
2
Введіть повне ім'я (буде відображатись у профілі), email (використовується для входу та відновлення паролю) та пароль (мінімум 8 символів, рекомендуємо використовувати літери, цифри та спецсимволи).
3
Натисніть кнопку реєстрації. На вказаний email буде відправлено лист з посиланням для верифікації. Перевірте також папку "Спам", якщо лист не прийшов протягом кількох хвилин.
4
Перейдіть за посиланням з листа. Після успішної верифікації ваш акаунт буде активовано, і ви зможете увійти в систему.

Вхід в акаунт

Для входу відкрийте сторінку /login та введіть email і пароль, які ви використали при реєстрації. Якщо у вас увімкнена двофакторна автентифікація (2FA), система додатково попросить ввести 6-значний код з вашого додатку-автентифікатора.

Після успішного входу ви будете автоматично перенаправлені на дашборд. Сесія зберігається протягом 7 днів — вам не потрібно входити повторно при кожному відвідуванні.

Відновлення паролю

Якщо ви забули пароль:

1
На сторінці входу натисніть "Забули пароль?".
2
Введіть email вашого акаунту. На пошту прийде лист з одноразовим посиланням для скидання паролю.
3
Перейдіть за посиланням та задайте новий пароль. Посилання має обмежений термін дії — використайте його якнайшвидше.

Демо-рахунок та реальний рахунок

Yamato надає кожному користувачу два окремих рахунки:

  • Демо-рахунок (Demo) — при реєстрації вам автоматично нараховується віртуальний баланс. Ви можете торгувати, створювати ботів, розміщувати ордери — все працює так само як на реальному рахунку, але без фінансового ризику. Ідеально для навчання та тестування стратегій.
  • Реальний рахунок (Real) — для торгівлі справжніми криптовалютами. Вимагає підключення до Binance через API ключі. Операції виконуються безпосередньо на вашому рахунку Binance.

Переключення між рахунками відбувається у верхній частині дашборду або в налаштуваннях профілю. Баланси демо та реального рахунків повністю незалежні один від одного.

Порада: Рекомендуємо спочатку освоїти всі функції на демо-рахунку. Тестуйте стратегії, створіть кілька ботів у Paper-режимі та переконайтесь, що розумієте механіку платформи, перш ніж переходити до реальної торгівлі.

Faucet — поповнення демо-балансу

Якщо ваш демо-баланс вичерпався, скористайтесь функцією Faucet для отримання додаткових віртуальних коштів. Це дозволить продовжити тестування без обмежень. Faucet доступний на сторінці дашборду.

Дашборд

Дашборд (/dashboard) — це головна робоча панель трейдера. Тут зібрані всі ключові інструменти для моніторингу ринку, аналізу графіків та прийняття торгових рішень. Дашборд є першою сторінкою, яку ви бачите після входу в систему.

TradingView графік

Центральний елемент дашборду — інтерактивний графік на базі бібліотеки TradingView. Графік отримує дані безпосередньо від Binance в реальному часі через WebSocket-з'єднання, що забезпечує мінімальну затримку.

Можливості графіку:

  • Таймфрейми — переключайте між різними часовими інтервалами: 1m, 3m, 5m, 15m, 30m, 1h, 2h, 4h, 6h, 8h, 12h, 1d, 3d, 1w, 1M. Кожен таймфрейм показує свічки відповідного масштабу.
  • Індикатори — додавайте технічні індикатори прямо на графік: Moving Average (MA), Exponential MA (EMA), RSI, MACD, Bollinger Bands, Volume Profile, Stochastic, Ichimoku Cloud та десятки інших.
  • Інструменти малювання — лінії тренду, горизонтальні рівні підтримки/опору, канали, фібоначчі-рівні, прямокутники, стрілки та інші графічні елементи для розмітки графіку.
  • Торгові пари — переключайтесь між будь-якими парами, доступними на Binance (BTC/USDT, ETH/USDT, BNB/USDT тощо). Пошук по тікеру дозволяє швидко знайти потрібну пару.
  • Типи графіків — свічковий (candlestick), лінійний, барний, Heikin Ashi та інші типи відображення.

Віджети дашборду

Окрім графіку, дашборд містить додаткові віджети з корисною інформацією:

  • Ринкові ціни — таблиця з актуальними цінами основних криптовалют, зміною за 24 години (у відсотках та абсолютному значенні), обсягом торгів.
  • Баланс рахунку — поточний стан вашого демо або реального балансу.
  • Активні ордери — список ваших відкритих ордерів з можливістю швидкого скасування.
  • Стан ботів — короткий огляд активних торгових ботів та їх поточного прибутку.

Кастомізація (drag & resize)

Дашборд повністю кастомізується під ваш стиль торгівлі:

  • Перетягування (drag) — захопіть віджет за верхню частину та перетягніть в потрібне місце. Інші віджети автоматично перебудуються.
  • Зміна розміру (resize) — потягніть за край або кут віджета для зміни його розміру. Графік, наприклад, можна розтягнути на весь екран для детального аналізу.
  • Збереження — ваше розташування віджетів зберігається автоматично і відновлюється при наступному вході.
Порада: Для максимально зручної роботи розгорніть графік на більшу частину екрану, а допоміжні віджети розмістіть збоку або знизу. Так ви зможете одночасно аналізувати графік та контролювати ринкові дані.

Ринкові дані

На дашборді відображаються ключові метрики ринку в реальному часі:

  • Ціна — поточна ціна обраної торгової пари з точністю до центів
  • Зміна 24h — відсоткова та абсолютна зміна ціни за останні 24 години. Зелений колір означає зростання, червоний — падіння.
  • Обсяг 24h — загальний обсяг торгів за 24 години в базовій та котирувальній валюті
  • High / Low 24h — максимальна та мінімальна ціна за 24 години

Всі дані оновлюються автоматично без необхідності перезавантаження сторінки.

Портфель

Сторінка портфелю (/portfolio) надає повний огляд вашого фінансового стану на платформі. Тут ви бачите всі свої активи, їх вартість, розподіл та детальну історію операцій.

Загальний баланс

У верхній частині сторінки відображається ваш загальний баланс — сума всіх ваших активів у доларовому еквіваленті (USDT). Це включає як вільні кошти на рахунку, так і кошти, що знаходяться в активних позиціях та ордерах. Баланс перераховується автоматично при зміні ринкових цін.

Розподіл активів

Графічна візуалізація (діаграма) показує, як ваш портфель розподілений між різними активами. Це допомагає контролювати диверсифікацію — якщо занадто велика частка припадає на один актив, можливо, варто перебалансувати портфель для зменшення ризиків.

Список холдингів

Детальна таблиця з інформацією по кожному активу у вашому портфелі:

  • Назва та тікер — наприклад, Bitcoin (BTC), Ethereum (ETH)
  • Кількість — скільки одиниць цього активу ви тримаєте
  • Поточна ціна — актуальна ринкова ціна активу
  • Загальна вартість — кількість помножена на поточну ціну
  • Прибуток/Збиток (PnL) — різниця між поточною вартістю та ціною покупки, у відсотках та абсолютному значенні
  • Частка портфелю — який відсоток загального портфелю займає цей актив

Історія транзакцій

Повний лог всіх фінансових операцій на вашому рахунку:

  • Покупки — придбання криптовалюти (Market або Limit ордер)
  • Продажі — продаж криптовалюти
  • Операції ботів — автоматичні угоди, виконані торговими ботами
  • Поповнення — зарахування коштів на рахунок

Кожна транзакція містить дату, час, тип операції, актив, кількість, ціну та статус.

Порада: Регулярно перевіряйте розподіл портфелю. Здорова диверсифікація допомагає зменшити ризики. Уникайте ситуацій, коли більше 50% портфелю зосереджено в одному активі, якщо ви не впевнені в його перспективах.

Торгові боти

Торгові боти — це один з найпотужніших інструментів Yamato. Бот автоматично виконує торгові операції за заданими параметрами та стратегією, працюючи 24 години на добу, 7 днів на тиждень, навіть коли ви не онлайн.

Боти підключаються до Binance через API і можуть торгувати будь-якими доступними парами. Yamato надає зручний інтерфейс для створення, налаштування, моніторингу та управління ботами.

Створення нового бота

1
Перейдіть на сторінку /bots та натисніть кнопку створення нового бота. Відкриється форма з налаштуваннями.
2
Назва бота — задайте зрозумілу назву, наприклад "BTC Grid 1h" або "ETH DCA Daily". Це допоможе відрізняти ботів у списку.
3
Тип бота — оберіть стратегію торгівлі. Різні типи ботів використовують різні алгоритми для прийняття рішень про купівлю та продаж.
4
Торгова пара — оберіть пару, якою бот буде торгувати (наприклад BTC/USDT, ETH/USDT). Бот буде відстежувати ціну саме цієї пари.
5
Режим роботи — оберіть Paper (симуляція) або Live (реальна торгівля). Для Live-режиму потрібні API ключі Binance.
6
API ключі Binance (тільки для Live) — вкажіть API Key та API Secret вашого акаунту Binance. Створити ключі можна у налаштуваннях Binance в розділі API Management.
7
Сума інвестиції — вкажіть суму (в USDT), яку бот може використовувати для торгівлі. Бот не перевищить цю суму.
8
Торгові налаштування — додаткові параметри стратегії: розмір позиції, частота угод, умови входу/виходу та інші, залежно від типу бота.

Налаштування API ключів Binance

Для роботи ботів у Live-режимі потрібні API ключі Binance. Ось як їх створити правильно:

1
Увійдіть у свій акаунт Binance та перейдіть в Account > API Management.
2
Створіть новий API ключ. Задайте зрозумілу назву, наприклад "Yamato Bot".
3
Дозволи — увімкніть тільки "Enable Spot & Margin Trading". Категорично не вмикайте "Enable Withdrawals" — бот не потребує можливості виведення коштів.
4
Обмеження за IP (рекомендовано) — якщо ви знаєте IP-адресу сервера Yamato, додайте її в whitelist. Це гарантує, що ключ можна використовувати тільки з дозволеної адреси.
5
Скопіюйте API Key та Secret Key (Secret показується тільки один раз!) та вставте їх у форму створення бота на Yamato.
Увага: Ніколи не передавайте API ключі третім особам. Ніколи не вмикайте дозвіл на виведення коштів (Withdrawals) для ключів, які використовуються ботами. Якщо ви підозрюєте, що ключ скомпрометований — негайно видаліть його в налаштуваннях Binance та створіть новий.

Режими роботи ботів

  • Paper (паперова торгівля) — бот працює в режимі симуляції. Він відстежує реальні ринкові ціни та "виконує" угоди віртуально, але жодна реальна транзакція не відбувається. Це ідеальний спосіб протестувати стратегію перед запуском на реальні кошти. Результати Paper-торгівлі максимально наближені до реальних умов ринку.
  • Live (жива торгівля) — бот виконує реальні операції на вашому акаунті Binance через API. Кожна угода відображається як у Yamato, так і на Binance. Переконайтесь, що на вашому рахунку Binance достатньо коштів для торгівлі.
Порада: Завжди спочатку запускайте нового бота в Paper-режимі мінімум на кілька днів. Це допоможе оцінити ефективність стратегії та виявити можливі проблеми до того, як ви ризикуєте реальними грошима.

Моніторинг ботів

На сторінці /bots ви бачите список всіх ваших ботів з ключовою інформацією:

  • Статус — активний (працює), на паузі або зупинений
  • Торгова пара — на якій парі працює бот
  • Прибуток (PnL) — загальний результат бота з моменту запуску
  • Кількість угод — скільки торгових операцій виконав бот
  • Режим — Paper або Live
  • Час роботи — як довго бот працює

Детальна статистика бота

Для кожного бота доступна окрема сторінка детальної статистики (/bot-stats/:id), яка включає:

  • Графік прибутку за часом (equity curve)
  • Список всіх угод з цінами входу та виходу
  • Середній прибуток/збиток на угоду
  • Відсоток успішних угод (win rate)
  • Максимальна просадка (max drawdown)
  • Загальний обсяг торгів

Управління ботами

  • Активація/Пауза — запускайте або зупиняйте бота в будь-який момент. При паузі бот припиняє відкривати нові позиції, але може дочекатися закриття поточних.
  • Дублювання — створіть копію бота з тими самими налаштуваннями. Корисно для запуску кількох однакових ботів на різних парах або для A/B-тестування.
  • Редагування — змініть налаштування бота (торгову пару, параметри стратегії, суму інвестиції). Деякі зміни вимагають зупинки бота.
  • Видалення — видалення бота незворотне. Переконайтесь, що бот не має відкритих позицій перед видаленням.

Налаштування відображення (Display Settings)

Кожен бот має налаштування відображення, які дозволяють кастомізувати інформаційну панель бота: які дані показувати на картці бота, формат відображення числових даних, видимість окремих метрик. Це допомагає зосередитись на найважливішій інформації для кожного конкретного бота.

Підписка на бота

Yamato дозволяє підписуватися на публічних ботів інших трейдерів. Це означає, що ви можете копіювати стратегії успішних трейдерів та автоматично повторювати їх угоди на своєму рахунку. Перед підпискою обов'язково перевірте статистику бота та його історичні результати.

Ордери

Ордер — це інструкція на купівлю або продаж криптовалюти за певних умов. Yamato підтримує кілька типів ордерів, кожен з яких має свої переваги та сценарії використання.

Ринковий ордер (Market Order)

Ринковий ордер — найпростіший тип. Він виконується миттєво за поточною найкращою доступною ціною на ринку.

  • Коли використовувати: Коли вам потрібно негайно купити або продати актив, і точна ціна менш важлива, ніж швидкість виконання.
  • Переваги: Гарантоване виконання (за умови достатньої ліквідності).
  • Недоліки: Кінцева ціна може дещо відрізнятися від тієї, що ви бачили на екрані (прослизання/slippage), особливо на волатильному ринку або при великих обсягах.

Лімітний ордер (Limit Order)

Лімітний ордер виконується тільки коли ціна досягне вказаного вами рівня або кращого.

  • Лімітна покупка — встановіть ціну нижче поточної. Ордер виконається, коли ціна впаде до вашого рівня. Наприклад, якщо BTC коштує $60,000, а ви хочете купити за $58,000 — створіть лімітну покупку на $58,000.
  • Лімітний продаж — встановіть ціну вище поточної. Ордер виконається, коли ціна зросте до вашого рівня.
  • Переваги: Повний контроль над ціною виконання. Ви завжди отримаєте вказану ціну або кращу.
  • Недоліки: Немає гарантії виконання — якщо ціна не досягне вашого рівня, ордер залишиться відкритим.

Стоп-лосс (Stop Loss)

Стоп-лосс — це захисний ордер, який автоматично продає актив, коли ціна падає до вказаного рівня. Основна мета — обмежити потенційні збитки.

  • Приклад: Ви купили ETH за $3,000. Встановіть стоп-лосс на $2,700 (зниження на 10%). Якщо ціна впаде до $2,700, ETH буде автоматично проданий, і ваш максимальний збиток складе 10%.
  • Рекомендація: Завжди встановлюйте стоп-лосс для кожної позиції. Визначте максимально допустимий збиток заздалегідь.

Тейк-профіт (Take Profit)

Тейк-профіт — ордер для автоматичної фіксації прибутку при досягненні цільової ціни.

  • Приклад: Ви купили BTC за $60,000 та хочете зафіксувати прибуток при зростанні до $66,000 (10%). Встановіть тейк-профіт на $66,000 — коли ціна досягне цього рівня, BTC буде автоматично проданий з прибутком.
  • Рекомендація: Встановлюйте реалістичні цілі на основі технічного аналізу (рівні опору, фібоначчі тощо).

Управління ордерами

Усі ваші відкриті ордери відображаються на дашборді та на сторінці ордерів. Для кожного ордера доступна наступна інформація та дії:

  • Тип ордера, торгова пара, напрямок (Buy/Sell)
  • Ціна та кількість
  • Статус: відкритий, виконаний, скасований
  • Час створення та час виконання
  • Кнопка скасування для відкритих ордерів
Порада: Для кожної позиції використовуйте комбінацію стоп-лосс та тейк-профіт. Це створює "зону" торгівлі, де ви заздалегідь визначаєте максимальний збиток та цільовий прибуток. Популярне співвідношення Risk/Reward — 1:2 або 1:3 (наприклад, ризикуєте 5%, цілитесь на 10-15%).
Увага: Торгівля криптовалютами пов'язана з високим ризиком. Ніколи не інвестуйте більше, ніж готові втратити. Стоп-лосс ордери не гарантують виконання за точною ціною в умовах екстремальної волатильності (gap).

Редагування профілю

На сторінці профілю ви можете оновити наступну інформацію:

  • Аватар — завантажте фото або зображення для персоналізації
  • Email — зміна email потребує повторної верифікації нової адреси
  • Пароль — для зміни потрібно ввести поточний пароль та новий (мінімум 8 символів)

Двофакторна автентифікація (2FA / TOTP)

Двофакторна автентифікація додає другий рівень захисту. Навіть якщо хтось дізнається ваш пароль, він не зможе увійти без коду з вашого телефону.

1
Перейдіть в розділ Безпека на сторінці профілю та натисніть "Увімкнути 2FA".
2
Система згенерує QR-код. Відскануйте його додатком-автентифікатором на вашому смартфоні: Google Authenticator, Microsoft Authenticator, Authy або інший TOTP-сумісний додаток.
3
Додаток покаже 6-значний код, який змінюється кожні 30 секунд. Введіть поточний код у поле підтвердження на сайті.
4
Після підтвердження 2FA буде активована. Тепер при кожному вході система буде запитувати 6-значний код з вашого додатку на додаток до паролю.
Увага: При налаштуванні 2FA обов'язково збережіть секретний ключ (або зробіть скріншот QR-коду) у безпечному місці. Якщо ви втратите доступ до додатку-автентифікатора (наприклад, зламається телефон), цей ключ буде єдиним способом відновити доступ до 2FA.

Passkeys (WebAuthn)

Yamato підтримує сучасний стандарт WebAuthn (Passkeys) — це найбезпечніший метод автентифікації, доступний на сьогодні. Passkeys використовують криптографічні ключі, що зберігаються на вашому пристрої.

Як це працює:

  • Біометрія — вхід за відбитком пальця (Touch ID, fingerprint) або розпізнаванням обличчя (Face ID)
  • Апаратний ключ — USB-ключ безпеки (YubiKey, Titan Key тощо)
  • PIN пристрою — PIN-код вашого комп'ютера або телефону (Windows Hello, macOS Touch ID)

Щоб додати Passkey, перейдіть в розділ безпеки профілю, натисніть "Додати Passkey" та слідуйте інструкціям браузера. Ви можете додати кілька Passkeys для різних пристроїв.

Часті питання (FAQ)

Чи безпечно зберігати API ключі Binance на платформі?
Так, API ключі зберігаються у зашифрованому вигляді на сервері. Однак для максимальної безпеки ми рекомендуємо: 1) Створювати ключі з мінімально необхідними дозволами — тільки "Enable Spot & Margin Trading", без "Enable Withdrawals". 2) Обмежити доступ ключа за IP-адресою у налаштуваннях Binance. 3) Регулярно перевіряти активність ключів та видаляти невикористовувані. Навіть якщо хтось отримає доступ до вашого ключа, він не зможе вивести кошти з вашого рахунку, якщо дозвіл на виведення вимкнений.
Що таке Paper-торгівля і чим вона відрізняється від реальної?
Paper-торгівля (паперова торгівля) — це режим симуляції, де бот або ви особисто торгуєте на віртуальних коштах. Бот відстежує реальні ринкові ціни від Binance та "виконує" угоди віртуально, фіксуючи результати так, ніби вони реальні. Різниця з реальною торгівлею: 1) Немає проблем з ліквідністю (у Paper завжди є контрагент). 2) Немає прослизання (slippage). 3) Немає комісій біржі. Тому реальні результати можуть дещо відрізнятися від Paper. Але загалом Paper — відмінний спосіб протестувати стратегію перед тим, як ризикувати реальними грошима.
Як отримати демо-кошти?
При реєстрації кожен користувач отримує початковий демо-баланс автоматично. Якщо ваш демо-баланс вичерпався (наприклад, внаслідок збиткових угод), скористайтесь функцією Faucet на дашборді для поповнення віртуальних коштів. Faucet можна використовувати кілька разів, що дозволяє продовжувати навчання без обмежень.
Як підключити Telegram-сповіщення?
Перейдіть на сторінку профілю (/profile) та знайдіть секцію Telegram у налаштуваннях. Натисніть кнопку прив'язки — відкриється Telegram-бот Yamato. Натисніть /start у боті та підтвердіть прив'язку. Після успішного з'єднання ви почнете отримувати сповіщення про торгові операції, виконання ордерів, спрацювання ботів та інші важливі події. Ви можете відв'язати Telegram у будь-який момент з тієї ж сторінки профілю.
Чи можу я мати кілька ботів одночасно?
Так, ви можете запускати кілька ботів паралельно. Кількість одночасно активних ботів залежить від вашого плану підписки — безкоштовний план має обмеження, а платні плани дозволяють запускати більше ботів. Кожен бот може працювати на окремій торговій парі та з власними налаштуваннями. Це дозволяє диверсифікувати стратегії та торгувати на кількох ринках одночасно.
Що робити, якщо я забув пароль?
На сторінці входу (/login) натисніть "Забули пароль?" та введіть email вашого акаунту. На пошту прийде лист з одноразовим посиланням для скидання паролю. Перейдіть за посиланням та задайте новий пароль. Важливо: посилання має обмежений термін дії, тому використайте його якнайшвидше. Якщо лист не прийшов — перевірте папку "Спам". Якщо проблема зберігається, зверніться до підтримки через Telegram.
Які криптовалюти підтримує платформа?
Yamato підтримує всі торгові пари, доступні на біржі Binance. Це включає: основні криптовалюти (BTC, ETH, BNB, SOL, XRP, ADA та інші), стейблкоїни (USDT, USDC, BUSD), а також сотні альткоїнів. Графіки та ринкові дані оновлюються в реальному часі через WebSocket-з'єднання з Binance. Нові пари, що з'являються на Binance, стають автоматично доступними на Yamato.
Чи можу я торгувати з мобільного пристрою?
Так, Yamato має адаптивний дизайн та оптимізований мобільний інтерфейс. Всі основні функції доступні на смартфоні: перегляд графіків, моніторинг ботів, управління портфелем, створення ордерів. Нижня мобільна навігація забезпечує швидкий доступ до ключових розділів. Для найзручнішої роботи рекомендуємо використовувати десктопну версію для аналізу графіків та створення ботів, а мобільну — для моніторингу та швидких операцій.
Як працює переключення між демо та реальним рахунком?
Демо та реальний рахунки повністю незалежні один від одного. Кожен має свій окремий баланс, портфель, історію транзакцій та активні ордери. Переключення відбувається в налаштуваннях профілю або на дашборді. При переключенні всі дані відповідного рахунку завантажуються автоматично. Боти також прив'язані до типу рахунку — Paper-боти працюють на демо, Live-боти на реальному.
Що робити, якщо бот працює зі збитком?
Перш за все, проаналізуйте статистику бота на сторінці /bot-stats — перевірте win rate, середній PnL на угоду, максимальну просадку. Якщо збиток системний: 1) Зупиніть бота. 2) Перевірте чи правильно налаштовані параметри стратегії. 3) Проаналізуйте ринкові умови — можливо, стратегія не підходить для поточного стану ринку (тренд, флет). 4) Протестуйте змінені параметри спочатку в Paper-режимі. Пам'ятайте: жодна стратегія не працює ідеально 100% часу. Збитки — нормальна частина торгівлі, головне — щоб загальний результат був позитивним.
Як зв'язатись з підтримкою?
Найшвидший спосіб зв'язатись з командою Yamato — через офіційний Telegram-канал. Посилання на всі офіційні канали зв'язку знаходяться на сторінці /community. При зверненні опишіть проблему якомога детальніше: що саме не працює, коли це почалось, які кроки ви вже спробували. Скріншоти також допоможуть швидше розібратися в ситуації.
